# Component Rules

These rules are implementation-facing UX guidelines for a desktop SaaS design system.

## accordion

**Use it when**  
Reveal or hide related content inline.

**Do not use it when**  
Do not hide essential information users must always compare side by side.

**Best practices**
- Use only semantic tokens from theme and global layers.
- Respect state priority: disabled > focused > pressed > hovered > default.
- Keep slots consistent with the component API and avoid ad-hoc children.

**Anti-patterns**
- Do not hardcode colors, spacing, shadows, or focus styles.
- Do not invent new states or variants not present in the API.

- Do not invent new states or variants not present in the API.

## tooltip

**Use it when**  
Show contextual feedback, additional context, or blocking confirmation content.

**Do not use it when**  
Do not hide critical workflow steps behind transient surfaces without an alternative.

**Best practices**
- Use only semantic tokens from theme and global layers.
- Respect state priority: disabled > focused > pressed > hovered > default.
- Keep slots consistent with the component API and avoid ad-hoc children.
